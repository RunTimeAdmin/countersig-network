TESTNET — Q3 2026

Decentralized Identity
for Autonomous AI Agents

Countersig Network is an open protocol that gives AI agents verifiable on-chain identities, cryptoeconomically-secured reputation scores, and trustless agent-to-agent authentication — without a central authority.

Register an Agent View on GitHub

The Problem

AI agents can execute tasks, transact on-chain, and communicate via A2A protocols. But they cannot prove who they are. Without verifiable identity, every agent interaction requires pre-existing trust — which does not scale.

No Verifiable Identity

Any agent can claim to be any other agent. There is no cryptographic proof of ownership.

No Portable Reputation

Reputation is siloed per platform. An agent starts from zero on every new marketplace.

No Accountability

When an agent causes harm, there is no stake to slash and no operator to hold responsible.



The Protocol

Three on-chain registries, one staking core, and a TypeScript SDK.

LAYER 3

Off-Chain Verification

Agents authenticate peer-to-peer using Ed25519 PKI challenge-response. On-chain registries serve as the source of truth for public keys and reputation scores.

LAYER 2

Decentralized Identity Layer

The did:countersig method resolves on-chain state to W3C-compliant DID Documents. Any resolver can reproduce the DID hash without querying the contract.

LAYER 1

On-Chain State (EVM)

Three UUPS-upgradeable Solidity registries: Identity (DID anchoring), Reputation (6-factor score), and Staking (bonding + slashing). All state is public and composable.

How It Works

From zero to a live, reputation-tracked AI agent in five steps.

1

Generate an Ed25519 keypair

Your agent's cryptographic identity lives off-chain. The public key gets registered on-chain; the private key signs challenges when peers ask your agent to prove who it is. No central authority issues or controls it.

2

Stake $CSIG and register your DID

Deposit a minimum $CSIG bond, then call registerAgent(agentAddress, ed25519PubKey). The contract computes your didHash from the chain ID and address, stores the public key, and emits an event. Your DID — did:countersig:<chainId>:<address> — is now globally resolvable. The stake creates skin in the game: misbehavior leads to slashing and permanent loss of the bond.

3

Do work — with your DID on every action

Include agent_did in every audit call to CounterAudit. Each action is sealed into a tamper-evident packet. CounterAudit queries your live on-chain identity and reputation score at seal time and embeds both inside the AES-GCM-sealed, RFC 3161-timestamped record. The reputation score is frozen at the moment of the action — not retroactively updated.

4

The oracle computes your reputation

An oracle network watches on-chain signals — fee volume, task attestations, registration age, external trust sources — and writes a 6-factor score (0–100) to the CountersigReputation contract every epoch. Any smart contract or off-chain consumer can call getTotalScore(didHash) or meetsThreshold(didHash, minScore). The computation is off-chain; the result is on-chain.

5

Reputation unlocks trust — and more work

Peers that require proof of identity issue an Ed25519 challenge. Your agent signs it. The verifier resolves your public key from the contract and checks the signature without contacting any third party. Higher reputation means peers set lower barriers — your agent gets access to higher-value work, which generates more fees, which increases your reputation further. The loop is self-reinforcing and Sybil-resistant by design.

Why the sealed packet matters

Each CounterAudit packet freezes the agent's reputation at the moment of the action. If an agent is slashed next month, every audit record from before the slash still shows what their reputation was then. The historical record does not rewrite itself — the RFC 3161 timestamp and AES-GCM seal together guarantee it.

From zero to a registered, reputation-tracked AI agent on Sepolia in about 10 minutes. You need Node.js 18+, a wallet with Sepolia ETH, and an Alchemy or Infura RPC URL.

1

npm install @countersig/protocol-sdk ethers
2

Get testnet $CSIG

import { ethers } from 'ethers';

const provider = new ethers.JsonRpcProvider(process.env.RPC_URL);
const signer   = new ethers.Wallet(process.env.OPERATOR_PRIVATE_KEY, provider);
const csig     = new ethers.Contract(
  '0x6d5E311e821c3e279dBe9833F8e33828f7716FA8',
  ['function faucet() external'],
  signer

await csig.faucet();

Your agent's cryptographic identity. The public key is stored on-chain. The private key never leaves your environment — store it securely, it cannot be recovered.

4

Approve and register on-chain

Two transactions: approve the staking contract to spend CSIG, then register. The contract computes your didHash from your address and chain ID and emits AgentRegistered.

const IDENTITY_ADDRESS = '0xD738A4cBe525d214f86059A8328786f072D6fbe1';

console.log('Registered:', `https://sepolia.etherscan.io/tx/${txHash}`);
5

Verify registration and check score

const identity = await verifier.getIdentity(agent.did);

6

Sign a challenge (A2A authentication)

When a peer agent asks your agent to prove who it is, sign the challenge with your Ed25519 key. The verifier resolves your public key directly from the contract — no third party involved.

const myAgent = new CountersigAgent({
  privateKey:   process.env.AGENT_ED25519_SEED,
  agentAddress: signer.address,
  chainId:      11155111,
});

// Peer issues a challenge → your agent signs it
const signature = myAgent.signChallenge(challenge.payload);

// Peer verifies: signature + on-chain pubkey + reputation threshold
const valid   = await verifier.verifySignature(myAgent.did, challenge.payload, signature);
const trusted = await verifier.meetsThreshold(myAgent.did, 60); // view call, no gas

Enriched Fields in Every Sealed Packet

FieldTypeDescription
agent_didstringThe W3C DID: did:countersig:<chainId>:<address>
agent_did_hashhexOn-chain index key — any party can verify it matches the DID independently
agent_chain_idnumberEVM chain ID (e.g. 11155111 for Sepolia)
agent_reputation_score0–100Total score at the moment of ingest — frozen in the seal, not retroactively updated

agent_enrichment_errorstringPresent on failure: invalid_did_format · not_registered · rpc_timeout · rpc_error
Enrichment is never a blocking dependency. If the RPC times out or the DID is unregistered, agent_identity_verified is set to false and the packet seals normally.

Trust Gate Pattern

Block agents below a reputation threshold before their action is audited — a single read-only view call, no gas required.

const verifier = new CountersigVerifier({ rpcUrl, addresses, chainId: 11155111 });

async function handleAgentAction(agentDid: string, action: object) {
  const meets = await verifier.meetsThreshold(agentDid, 30);
  if (!meets) throw new Error(`Agent does not meet minimum reputation threshold`);
  return auditAction(agentDid, action); // enrich + seal
}

withAudit middleware (works with any async function)

import { CountersigAgent, CountersigVerifier } from '@countersig/protocol-sdk';

const myAgent = new CountersigAgent({
  privateKey:   process.env.AGENT_ED25519_SEED!,

});

async function withAudit<T>(action: string, fn: () => Promise<T>, meta?: object): Promise<T> {
  const result = await fn();
  // fire-and-forget — does not block the critical path
  fetch('https://api.counteraudit.io/v1/audit/ingest', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${process.env.CA_API_KEY}`, 'Content-Type': 'application/json' },
    body: JSON.stringify({ connector_id: 'my-agent', agent_did: myAgent.did, raw_event: { action, ...meta } }),
  }).catch(console.error);
  return result;
}

Verify a peer agent before trusting its output

const verifier = new CountersigVerifier({ rpcUrl, addresses, chainId: 11155111 });

async function trustAgent(did: string, payload: string, sig: string): Promise<boolean> {
  const sigValid = await verifier.verifySignature(did, payload, sig);
  const repOk    = await verifier.meetsThreshold(did, 50);
  return sigValid && repOk;
}

A deterministic 6-factor score (0–100) computed by the oracle network every 24-hour epoch and written on-chain. Sybil-resistant: Age and Fee Activity require sustained real economic activity.

Factor Details

FactorMaxFormulaSource
Fee Activity30 ptsmin(30, floor(totalFeesUSD / 100))On-chain transaction volume. $3,000+ in fees reaches max. Hardest to fake — requires sustained economic participation.
Success Rate25 ptsfloor(successRate × 25)Cryptographic task attestations submitted by counterparties. 100% success rate = 25 pts. Will be sourced from CounterAudit verified packets in Phase 2.
Registration Age20 ptsmin(20, floor(log₂(days+1) × 4))Logarithmic — levels off around day 31. Prevents idle old agents from dominating.
External Trust15 ptsfloor(externalScore / 100 × 15)SAID Protocol / Gitcoin Passport. Bridges existing verified identity credentials.
Community5 ptsmax(0, 5 − flags × 2)Deducted per unresolved community flag. Unflagged agents start at 5.
Trust Propagation5 ptsOracle-computedNetwork effects — agents trusted by high-reputation peers propagate a fraction of that trust.

Age Ramp-Up

A new agent cannot be trusted at the same level as one with months of economic activity. The logarithmic curve prevents instant reputation but rewards consistency.

Day 0 (just registered)0 pts
Day 14 pts
Day 38 pts
Day 712 pts
Day 1516 pts

New agent baseline: ~5 points (community score only). Reaching 50+ requires weeks of sustained activity. The 90+ range requires months. You cannot buy reputation instantly — you earn it.

Slashing Resets Everything

When an agent is slashed, zeroReputation(didHash) sets all six factor scores to zero. The agent status becomes Slashed — terminal, no further transitions. The stake is burned/distributed. A slashed identity cannot be reactivated; the operator must register a fresh address and start from zero. This finality is what makes the score meaningful.

// Any Solidity contract can gate on reputation
interface ICountersigReputation {
    function getTotalScore(bytes32 didHash) external view returns (uint8);
    function meetsThreshold(bytes32 didHash, uint8 min) external view returns (bool);
}

contract MyProtocol {
    ICountersigReputation rep = ICountersigReputation(0x0613C561C5003D7948Ea09dE2C1895965A5c3F27);

    function onlyTrustedAgent(bytes32 didHash) internal view {
        require(rep.meetsThreshold(didHash, 50), "insufficient reputation");
    }
}
